# Corp Persona Screen Check

목적: **20~60세 평범한(전문가가 아닌) 대한민국 직장인**에 가까운 합성 페르소나 N명을 뽑아, 각자의 시점에서 사용자가 전달한 화면 캡처(1장 또는 연속된 여러 장)를 **처음 볼 때** 어떻게 이해하는지 think-aloud 방식으로 한국어 시뮬레이션한다.

**핵심: 이 페르소나는 UX/디자인/개발 전문가가 아니다.** 화면을 "잘 아는 사람"이 아니라 "그냥 회사 다니는 보통 사람"이 처음 마주쳤을 때의 반응을 재현하는 것이 목적이다.

## 알려진 제약 (반드시 사용자에게 상기시킬 것)

Nemotron-Personas-Korea에는 **회사 규모(대기업/중소기업) 필드가 없다.** `occupation`(직업명 텍스트)과 `education_level`만 있다. 따라서 "직장인" 세그먼트는 **근사(proxy) 필터**로만 만들 수 있다:

- KSCO 대분류 기준 화이트칼라 직군(관리자·전문가·사무 종사자)으로 occupation 제한
- **UX/디자인/개발/데이터분석 등 화면 리터러시가 이미 높은 전문 직군은 기본적으로 제외** — 안 그러면 "비전문가 반응"이라는 목적과 어긋난다. 근거: `references/corp-proxy-keywords.md`의 EXPERT_EXCLUDE 목록
- `education_level`을 "대학교 졸업" 이상으로 제한

정밀 필터가 아니므로, 결과에는 항상 "근사 필터 기반" 임을 명시한다.

## 워크플로우

### Step 0: 페르소나 소스 확인 (매번 먼저 할 것)

`data/sample_personas.json` 파일이 있는지 확인한다.

- **있으면** → 이걸 페르소나 소스로 쓴다. huggingface.co 접속이나 다운로드가 전혀 필요 없다. 이 파일 안의 페르소나 카드(보통 15명 내외) 중에서 화면 성격에 맞게 5명 안팎을 골라 쓴다(연령대·직군이 고르게 섞이도록).
- **없으면** → 사용자에게 이 파일이 없다는 것과 `data/README.md`의 생성 절차(huggingface.co 접속 가능한 로컬/Colab 필요)를 안내한다. 그 자리에서 임의로 페르소나를 지어내려면 **반드시 "실제 Nemotron-Personas-Korea 데이터 기반이 아닌 가상 인물"임을 결과에 명시**한다.

### (선택) 실시간 재검색 — huggingface.co 접속 가능한 환경에서만

조직 네트워크에 huggingface.co가 허용되어 있거나, 로컬/Colab에서 직접 돌리는 경우에만 해당:

```bash
python scripts/download.py          # 최초 1회, 캐시 준비
python scripts/search.py \
  --age-min 20 --age-max 60 \
  --large-corp-proxy \
  --diversity sex,age_band,occupation_root \
  --persona-types summary,professional \
  --n 5 \
  --out _workspace/personas.json
```

`--large-corp-proxy`는 기본적으로 전문 기술직(개발/디자인 등)을 제외한다. 이 전문직 제외를 끄고 싶으면 `--include-tech-roles` 추가. 옵션 상세: `references/filter-cookbook.md`. 직군 목록: `references/corp-proxy-keywords.md`.

### Step 1: 화면 캡처 수집

사용자는 검증하고 싶은 범위를 캡처해서 전달한다. **단일 이미지일 수도, 연속된 여러 장(플로우/시나리오)일 수도 있다.** 여러 장이면 사용자가 준 순서 그대로를 "화면 1 → 화면 2 → ..." 순서로 다룬다 (예시: 문자 메시지 앱에서 "글쓰기 → 카메라 아이콘 탭 → 사진 선택" 같은 3단계 플로우).

### Step 2: Think-aloud 평가

Step 0에서 확보한 페르소나 카드를 근거로 Claude(에이전트)가 직접 수행한다 — 스크립트 호출이 아니라 **에이전트 자신의 추론**:

1. 페르소나 카드(연령·직무·학력·전문성 서술)를 읽고 이 사람이 되어본다. 전문가처럼 매끄럽게 이해하지 말고, **정말 처음 보는 보통 회사원**처럼 반응한다.
2. 화면(들)을 순서대로, 페르소나가 실제로 눈으로 보는 순서(위→아래, 왼쪽→오른쪽, 눈에 띄는 것부터)로 훑는다.
3. `references/evaluation-guide.md`의 **관찰 → 생각 → 행동 → 요약** 4단계 think-aloud 형식으로, 화면마다, 페르소나마다 한국어로 기록한다.
4. 연속 캡처인 경우, 이전 화면에서의 행동/혼란이 다음 화면 이해에 영향을 주는지도 반영한다 (예: 1번 화면에서 아이콘 의미를 잘못 이해했으면 2번 화면에서도 그 오해가 이어짐).
5. 전체 페르소나를 종합해서 공통으로 걸리는 지점을 정리한다.

## 출력 형식

이모지와 서식을 적극 활용해 가독성을 높인다. 각 페르소나 × 각 화면마다 think-aloud 블록을 만들고, **마지막 종합은 결론(권장 개선)을 먼저, 근거는 그 아래에** 배치한다:

```markdown
### 👩🏻 페르소나 A: 26세 여, 인사팀 사원, 디지털 친숙도 높음

#### 화면 1
👀 관찰: [이 페르소나 눈에 가장 먼저 들어오는 것, 있는 그대로]
💭 생각: [그게 뭘 하는 건지 어떻게 추측하는지, 헷갈리는 부분 포함]
➡️ 행동: [다음에 뭘 누를 것 같은지]
📝 요약: [한 줄 정리]

#### 화면 2
...

---

### 👨🏻 페르소나 B: 52세 남, 인사담당자, 경기
...

---
## 종합

### ⚠️ 권장 개선 (Top 3)
1. **[가장 시급한 개선안]** — 한 줄 설명
2. **[두 번째 개선안]** — 한 줄 설명
3. **[세 번째 개선안]** — 한 줄 설명

### 💡 왜 이렇게 판단했나
- (1번 근거) 페르소나 A, C가 공통으로 화면 1에서 ○○를 헷갈려함
- (2번 근거) ...
- (3번 근거) ...

### 🔍 참고: 페르소나별로만 갈리는 지점
- (우선순위는 낮지만 특정 세그먼트에서만 나타난 이슈)
```

**성별 이모지**: 남 👨🏻 / 여 👩🏻 (필요시 연령대 뉘앙스를 위해 👩🏻‍💼 등 변형 가능, 과하게 쓰지 않는다). **화면 라벨**은 항상 👀 관찰 / 💭 생각 / ➡️ 행동 / 📝 요약 순서 고정.

**종합 섹션 순서 고정**: ⚠️ 권장 개선(Top 3, 결론 먼저) → 💡 근거 → 🔍 참고(페르소나별 세부 차이). 권장 개선은 딱 3개만 — 그 이상 나열하면 우선순위가 흐려진다. 근거 없는 권장 개선(일반론)은 넣지 않는다.

## 주의

- **PII 없음**: 합성 데이터. 출처 표기 필수(CC BY 4.0): "NVIDIA Nemotron-Personas-Korea (CC BY 4.0)"
- 평가는 페르소나 텍스트에 근거한 **정성적 시뮬레이션**이며 실제 사용자 테스트를 대체하지 않는다.
- **전문가처럼 매끄럽게 이해시키지 말 것** — think-aloud의 가치는 "헷갈림"과 "오해"를 그대로 드러내는 데 있다. 모든 화면을 다 이해한 것처럼 쓰면 이 스킬의 목적이 무의미해진다.
- 근사 필터 결과가 너무 적으면 `--include-tech-roles`를 켜거나 occupation 화이트리스트를 완화.
