# "대기업 직장인" 근사(Proxy) 필터 근거

Nemotron-Personas-Korea에는 회사 규모(대기업/중소기업/스타트업) 필드가 없다. 대신 다음 두 축으로 근사한다.

## 1. occupation 화이트리스트 (KSCO 대분류 1·2·3 기준)

`--large-corp-proxy` 플래그는 아래 키워드 중 하나라도 occupation에 포함되면 통과시킨다 (OR 매칭).
관리자·전문가·사무 종사자 대분류에 속하는 대표 직군 위주로 구성했고, 서비스/판매/기능/단순노무/농림어업/군인은 제외했다.

```
관리자/임원: 관리자, 임원, 팀장, 본부장, 지점장, 부장, 이사

전문가(사무/기술): 개발자, 엔지니어, 연구원, 설계, 디자이너,
  회계사, 세무사, 변호사, 감정평가사, 노무사, 경영지도사,
  컨설턴트, 애널리스트, 마케터, 마케팅, 브랜드매니저, 기획자,
  PM, 프로덕트매니저, 데이터분석가, 통계, 번역가, 통역사,
  홍보, PR, IR담당자, 감사, 품질관리, 구매관리자

사무 종사자: 사무원, 경리, 인사담당자, 총무, 구매담당,
  영업관리자, 비서, 행정사무원, 재무담당, 회계사무원
```

이 목록은 `scripts/search.py`의 `OCCUPATION_WHITELIST` 상수에 그대로 반영되어 있다. 필요하면 직접 추가/삭제해서 세그먼트를 조정한다.

## 2. education_level 하한

기본적으로 `education_level`이 "대학교 졸업" 또는 "대학원 졸업"인 경우만 통과 (`--large-corp-proxy`에 포함된 기본값). 대기업 정규직 채용의 최소 학력 요건을 근사한 것으로, 예외(고졸 채용 등)를 배제하는 트레이드오프가 있음을 인지할 것.

## 3. 전문직 제외 (EXPERT_EXCLUDE) — "비전문가" 조건을 위해 필수

목적이 "이 화면을 처음 보는 평범한 직장인이 이해할 수 있는가"이기 때문에, 화면 리터러시가 이미 높은 직군은 **평가 결과를 왜곡시킨다.** 이런 사람들은 실제로는 대표성 없는 "이미 정답을 아는 평가자"가 되어버린다. `--large-corp-proxy`는 기본적으로 occupation에 다음 키워드가 있으면 제외한다:

```
개발자, 프로그래머, 엔지니어, 소프트웨어, UX, UI, 디자이너,
데이터분석가, 정보보안, IT, 시스템, 웹, 앱개발
```

이 제외를 끄고 싶으면(예: 오히려 개발자/디자이너의 반응이 궁금할 때) `--include-tech-roles` 플래그를 추가한다.

**주의**: "엔지니어"에는 화면과 무관한 직군(기계엔지니어, 전기엔지니어 등)도 포함될 수 있어 과다 제외될 수 있다. 결과가 너무 적으면 `EXPERT_EXCLUDE_KEYWORDS`(search.py 상단)를 더 구체적으로(예: "소프트웨어엔지니어"로) 좁혀서 조정한다.

## 4. (선택) 텍스트 키워드 가중

`professional_persona` / `career_goals_and_ambitions`에 다음 단어가 있으면 더 신뢰도 높은 후보로 간주할 수 있다 (`--keywords` 옵션에 추가로 넣어서 사용):

```
대기업, 본사, 계열사, 그룹사, 다국적기업, 상장기업, 글로벌기업, 사내
```

단, 이 필드는 자유 서술이라 커버리지가 낮다 — 필수 필터로 쓰면 결과가 지나치게 적어질 수 있어 **보조 신호**로만 권장한다.

## 한계 및 트레이드오프 요약

| 방법 | 장점 | 한계 |
|---|---|---|
| occupation 화이트리스트 | 커버리지 넓음 | 대기업/중소기업 구분 불가 (같은 "응용SW개발자"라도 스타트업일 수 있음) |
| education_level 하한 | 대기업 채용 조건과 상관관계 있음 | 학력만으로 규모 추정은 느슨한 근사 |
| 텍스트 키워드 | 직접적 신호 | 서술에 언급 안 된 경우가 다수라 재현율 낮음 |

세 가지를 조합해도 "정밀한 대기업 직장인 필터"는 아니다. 리포트/평가 결과를 공유할 때는 항상 "근사 세그먼트"임을 명시할 것.
